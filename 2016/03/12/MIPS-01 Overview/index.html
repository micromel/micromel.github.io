MIPS 01-Overview

Mar 12, 2016

What is Assembly Language?

Assembly language is a low-level language that lacks conveniences of methods, variables, arrays, and other abstract concepts. It is used to write programs that deal with CPU directly, which means that the instructions implemented are totally dependent on the type of processor. Nevertheless, assembly language is worth learning for every programmer who is interested in computer architecture.

What is MIPS?

MIPS is an assembly language with extended instructions to make it more programmer-friendly.

Data Representation in MIPS

In order to write MIPS programs, first we must understand how data is stored and computation is performed in the most basic level.

The lowest unit of data is a bit. A bit is a binary value representing either 1 or 0. Depending on the context, a binary value can be interpreted as “positive” and “negative”, “true” and “false”, or “on” and “off”; basically, it represents 2 possible states of the result of a computation.

A byte consists of 8 bits. In MIPS, 1 byte represents 1 ASCII character. Since a string is basically an array of characters, the string Hello, World! containing 13 ASCII characters (including “,”, “[space]”, and “!”) will take up 13 bytes in memory.

A word refers to the largest addressable unit of a register. Therefore, the length of a word depends on the number of bits that the processor is able to handle in one operation. A word would be 32 bits long on a 32-bit processor, whereas a word would be 64 bits long on a 64-bit processor. In MIPS, the data type that occupies a whole word is number.
